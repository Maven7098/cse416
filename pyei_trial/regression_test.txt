✦ I have updated regression_test.py to scale the exponential regression and the resulting
  plots from a 0–1 range to a 0–100 range (percentages), as requested.

  Key Changes Made:
   1. Data Scaling: Multiplying xdata, ydata_dem, and ydata_rep by 100 before performing
      the regression. This ensures the optimized parameters ($a$, $b$, and $c$) are
      compatible with a 0–100 scale for both axes.
   2. Regression Stability: Added initial guesses (p0) to curve_fit. This is critical
      when scaling up to 100, as the default starting values can lead to math overflows
      or poor convergence in exponential models.
   3. Plot Formatting:
       * Set fixed axis limits from 0 to 100.
       * Updated tick formatters to display whole numbers with a % symbol (e.g., "50%"
         instead of "0.5").
       * Switched to a smooth curve plotting method using np.linspace to ensure the
         regression line looks clean regardless of data density.
   4. Desmos Compatibility: The exported parameters in the JSON file and the console
      output are now directly usable in Desmos with the formula $y = a \cdot e^{b \cdot
      x} + c$, where $x$ and $y$ are values from 0 to 100.

  Note on Results:
  During testing, I observed that for some datasets (like Democratic votes in Georgia),
  the optimized parameters $a$ and $c$ may be very large (e.g., $\approx 5.8 \times
  10^5$). This occurs because the underlying data is nearly linear; the exponential model
  fits this by using a very large offset and a very small exponent to mimic a straight
  line. These parameters are mathematically correct and will work in Desmos, though they
  may appear counter-intuitive.
